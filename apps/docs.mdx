The documentation site is built with **Next.js 16** and **Fumadocs**, providing a modern, fast, and SEO-optimized documentation experience.

## Architecture

The docs app uses Next.js App Router with **Fumadocs MDX** for content management, automatic navigation generation, and built-in search.

## How It Works

### Content Structure

Documentation pages are stored as MDX files in `content/docs/`:

```
content/docs/
├── index.mdx              # Home page (/docs)
├── getting-started/
│   ├── index.mdx         # /docs/getting-started
│   ├── installation.mdx  # /docs/getting-started/installation
│   └── configuration.mdx # /docs/getting-started/configuration
├── apps/
│   ├── ui.mdx            # /docs/apps/ui
│   ├── api.mdx           # /docs/apps/api
│   └── docs.mdx          # /docs/apps/docs
└── guides/
    ├── authentication.mdx
    └── deployment.mdx
```

### Fumadocs Source Loader

The content is loaded via the source API:

```typescript title="lib/source.ts"
import { docs } from 'fumadocs-mdx:collections/server'
import { loader } from 'fumadocs-core/source'
import { lucideIconsPlugin } from 'fumadocs-core/source/lucide-icons'

export const source = loader({
  baseUrl: '/docs',
  source: docs.toFumadocsSource(),
  plugins: [lucideIconsPlugin()],
})
```

This provides:
- Type-safe page queries
- Automatic navigation tree
- Search index generation
- Frontmatter validation

### Page Generation

Docs pages are dynamically rendered:

```typescript title="app/docs/[[...slug]]/page.tsx"
import { source } from '@/lib/source'
import { notFound } from 'next/navigation'

export default async function DocsPage({
  params,
}: {
  params: { slug?: string[] }
}) {
  const page = source.getPage(params.slug)
  if (!page) notFound()

  const MDX = page.data.body

  return (
    <article>
      <h1>{page.data.title}</h1>
      <p>{page.data.description}</p>
      <MDX />
    </article>
  )
}
```

### Frontmatter Schema

Each MDX file includes frontmatter:

```mdx title="content/docs/example.mdx"
---
title: Page Title
description: Page description for SEO and preview cards
icon: Rocket
---

Page content goes here...
```

## Running the Docs Locally

### Development Server

```bash
# Start docs app only
nx dev docs

# Or start all apps
pnpm dev
```

The docs site runs on **http://localhost:3002**.

### File Watching

Fumadocs automatically watches MDX files and regenerates types:

```json title="package.json"
{
  "scripts": {
    "dev": "next dev --port 3002",
    "postinstall": "fumadocs-mdx"
  }
}
```

<Info>
  MDX files are processed at build time, so changes require a page refresh in development.
</Info>

  <Step title="Create MDX File">


## Search Functionality

Fumadocs includes built-in search:

```typescript title="app/api/search/route.ts"
import { source } from '@/lib/source'
import { createSearchAPI } from 'fumadocs-core/search/server'

export const { GET } = createSearchAPI('advanced', {
  indexes: source.getPages().map((page) => ({
    title: page.data.title,
    description: page.data.description,
    url: page.url,
    structuredData: page.data.structuredData,
  })),
})
```

Search features:
- **Fuzzy matching** - Typo-tolerant search
- **Instant results** - Client-side search with no API calls
- **Keyboard shortcuts** - `Cmd+K` or `Ctrl+K` to open
- **Result highlighting** - Matched terms highlighted

## Advanced Features

### OpenGraph Images

Automatic OG image generation:

```typescript title="app/og/docs/[...slug]/route.tsx"
import { source } from '@/lib/source'
import { ImageResponse } from 'next/og'

export async function GET(req: Request) {
  const page = source.getPage(params.slug)
  
  return new ImageResponse(
    <div>
      <h1>{page.data.title}</h1>
    </div>
  )
}
```

### LLM-Friendly Output

Export docs as plain text for LLM consumption:

```typescript title="app/llms-full.txt/route.ts"
import { source, getLLMText } from '@/lib/source'

export async function GET() {
  const pages = source.getPages()
  const content = await Promise.all(
    pages.map(page => getLLMText(page))
  )
  
  return new Response(content.join('\n\n---\n\n'))
}
```

Access at: `http://localhost:3002/llms-full.txt`
