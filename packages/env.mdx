# Environment Package

The `@beztack/env` package provides centralized, type-safe environment variable validation using T3 Env. It ensures required variables are present at build time and provides TypeScript autocomplete.

## Installation

The env package is required and included by default:

```bash Terminal
pnpm add @beztack/env
```

## Exports

The package provides separate configurations for different environments:

### API Environment (Nitro/Node.js)

```typescript
import { env } from "@beztack/env/api"
```

Server-side environment variables for the API:

- Database connection
- Better Auth secrets
- Payment provider credentials
- Email API keys
- OAuth secrets

### UI Environment (Vite/Browser)

```typescript
import { env } from "@beztack/env/ui"
```

Client-side environment variables (must be prefixed with `VITE_`):

- API URL
- Public API keys
- Feature flags
- Payment provider public keys

### Next.js Environment (Docs/Next.js)

```typescript
import { env } from "@beztack/env/nextjs"
```

Environment variables for Next.js applications.

## API Environment Variables

<Note>
  All API variables are validated at server startup. Missing required variables will throw an error.
</Note>

### Database

<ParamField path="DATABASE_URL" type="string" required>
  PostgreSQL connection string
  
  **Format:** `postgresql://user:password@host:port/database`
</ParamField>

### Authentication

<ParamField path="BETTER_AUTH_SECRET" type="string" required>
  Secret key for Better Auth sessions (min 32 characters)
</ParamField>

<ParamField path="BETTER_AUTH_URL" type="string" required>
  Base URL for authentication callbacks
  
  **Example:** `https://api.example.com`
</ParamField>

<ParamField path="APP_NAME" type="string" default="beztack">
  Application name for auth emails
</ParamField>

### Email (Resend)

<ParamField path="RESEND_API_KEY" type="string" required>
  Resend API key
</ParamField>

<ParamField path="RESEND_FROM_EMAIL" type="string" required>
  Verified sender email address
</ParamField>

<ParamField path="RESEND_FROM_NAME" type="string" required>
  Sender display name
</ParamField>

### Payments

<ParamField path="PAYMENT_PROVIDER" type="enum" default="polar">
  Active payment provider
  
  **Options:** `polar` | `mercadopago`
</ParamField>

<ParamField path="SUBSCRIPTION_MODE" type="enum" default="organization">
  Subscription entity type
  
  **Options:** `user` | `organization`
</ParamField>

<ParamField path="PAYMENTS_SUCCESS_URL" type="string" required>
  Redirect URL after successful payment
</ParamField>

<ParamField path="PAYMENTS_CANCEL_URL" type="string" required>
  Redirect URL after canceled payment
</ParamField>

#### Polar

<ParamField path="POLAR_ACCESS_TOKEN" type="string">
  Polar API access token (required if `PAYMENT_PROVIDER=polar`)
</ParamField>

<ParamField path="POLAR_WEBHOOK_SECRET" type="string">
  Polar webhook signature secret
</ParamField>

<ParamField path="POLAR_SERVER" type="enum" default="sandbox">
  **Options:** `sandbox` | `production`
</ParamField>

<ParamField path="POLAR_ORGANIZATION_ID" type="string">
  Polar organization UUID
</ParamField>

<ParamField path="POLAR_*_PRODUCT_ID" type="string">
  Product UUIDs for each plan tier and billing period:
  - `POLAR_BASIC_MONTHLY_PRODUCT_ID`
  - `POLAR_BASIC_YEARLY_PRODUCT_ID`
  - `POLAR_PRO_MONTHLY_PRODUCT_ID`
  - `POLAR_PRO_YEARLY_PRODUCT_ID`
  - `POLAR_ULTIMATE_MONTHLY_PRODUCT_ID`
  - `POLAR_ULTIMATE_YEARLY_PRODUCT_ID`
</ParamField>

#### Mercado Pago

<ParamField path="MERCADO_PAGO_ACCESS_TOKEN" type="string">
  Mercado Pago access token (required if `PAYMENT_PROVIDER=mercadopago`)
</ParamField>

<ParamField path="MERCADO_PAGO_WEBHOOK_SECRET" type="string">
  Mercado Pago webhook secret
</ParamField>

<ParamField path="MERCADO_PAGO_INTEGRATOR_ID" type="string">
  Mercado Pago integrator ID
</ParamField>

### Application

<ParamField path="APP_URL" type="string" required>
  Frontend application URL
  
  **Example:** `https://app.example.com`
</ParamField>

<ParamField path="NODE_ENV" type="enum" default="development">
  **Options:** `development` | `production` | `test`
</ParamField>

## Usage Examples

### API Server

```typescript apps/api/server/utils/db.ts
import { env } from "@beztack/env/api"
import postgres from "postgres"

const client = postgres(env.DATABASE_URL)

console.log("Connecting to:", env.NODE_ENV)
```

### UI Application

```typescript apps/ui/src/lib/api-client.ts
import { env } from "@beztack/env/ui"

const apiClient = axios.create({
  baseURL: env.VITE_API_URL,
})

export { apiClient }
```

### Conditional Logic

```typescript Type-safe Checks
import { env } from "@beztack/env/api"

if (env.PAYMENT_PROVIDER === "polar") {
  // TypeScript knows POLAR_ACCESS_TOKEN must exist
  const token = env.POLAR_ACCESS_TOKEN
  // Initialize Polar SDK
}

if (env.NODE_ENV === "production") {
  // Production-only code
}
```

## Type Safety

T3 Env provides autocomplete and type checking:

```typescript
import { env } from "@beztack/env/api"

// ✅ Valid - DATABASE_URL exists
const db = env.DATABASE_URL

// ❌ TypeScript error - typo
const db = env.DATABSE_URL

// ✅ Type inference
const mode: "development" | "production" | "test" = env.NODE_ENV
```

## Validation Rules

The package validates environment variables at startup:

### Custom Assertions

```typescript Source (packages/env/src/api.ts:106-166)
// Example: Payment provider validation
const activePaymentProvider = env.PAYMENT_PROVIDER

if (activePaymentProvider === "polar") {
  assertRequired("POLAR_ACCESS_TOKEN", env.POLAR_ACCESS_TOKEN)
  assertRequired("POLAR_WEBHOOK_SECRET", env.POLAR_WEBHOOK_SECRET)
  assertUuid("POLAR_ORGANIZATION_ID", env.POLAR_ORGANIZATION_ID)
  // ... all Polar-specific validations
}

if (activePaymentProvider === "mercadopago") {
  assertRequired("MERCADO_PAGO_ACCESS_TOKEN", env.MERCADO_PAGO_ACCESS_TOKEN)
}
```

### Schema Validation

```typescript Zod Schemas
import { z } from "zod"

const serverSchema = {
  DATABASE_URL: z.string().url(),
  BETTER_AUTH_SECRET: z.string().min(1),
  RESEND_FROM_EMAIL: z.string().email(),
  NODE_ENV: z.enum(["development", "production", "test"]),
  // ...
}
```

## Environment Files

Create `.env` files for each environment:

```bash .env.local (Development)
# Database
DATABASE_URL=postgresql://user:pass@localhost:5432/beztack

# Auth
BETTER_AUTH_SECRET=your-secret-min-32-chars-long
BETTER_AUTH_URL=http://localhost:3000
APP_NAME=My App

# Email
RESEND_API_KEY=re_xxxxxxxxxxxx
RESEND_FROM_EMAIL=noreply@example.com
RESEND_FROM_NAME=My App

# Payments
PAYMENT_PROVIDER=polar
SUBSCRIPTION_MODE=organization
PAYMENTS_SUCCESS_URL=http://localhost:5173/billing/success
PAYMENTS_CANCEL_URL=http://localhost:5173/billing

# Polar
POLAR_ACCESS_TOKEN=polar_xxxx
POLAR_WEBHOOK_SECRET=whsec_xxxx
POLAR_SERVER=sandbox
POLAR_ORGANIZATION_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
POLAR_BASIC_MONTHLY_PRODUCT_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
POLAR_PRO_MONTHLY_PRODUCT_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

# App
APP_URL=http://localhost:5173
NODE_ENV=development
```

```bash .env.ui (Frontend)
VITE_API_URL=http://localhost:3000
VITE_PAYMENT_PROVIDER=polar
VITE_SUBSCRIPTION_MODE=organization
```

<Warning>
  Never commit `.env` files to version control. Add them to `.gitignore`.
</Warning>

## Configuration Source

### API Configuration

```typescript packages/env/src/api.ts:35-103
export const env = createEnv({
  server: {
    DATABASE_URL: z.string().url(),
    BETTER_AUTH_SECRET: z.string().min(1),
    BETTER_AUTH_URL: z.string().url(),
    APP_NAME: z.string().min(1).default("beztack"),
    // ... full schema
  },
  runtimeEnv: process.env,
  emptyStringAsUndefined: true,
})
```

### UI Configuration

```typescript packages/env/src/ui.ts:8-40
export const env = createEnv({
  clientPrefix: "VITE_",
  client: {
    VITE_API_URL: z.string().url().default("http://localhost:3000"),
    VITE_BASE_PATH: z.string().default("/").optional(),
    VITE_MERCADO_PAGO_PUBLIC_KEY: z.string().default(""),
    VITE_PAYMENT_PROVIDER: z.enum(["polar", "mercadopago"]).default("polar"),
    VITE_SUBSCRIPTION_MODE: z.enum(["user", "organization"]).default("organization"),
  },
  runtimeEnv: import.meta.env,
  emptyStringAsUndefined: true,
})
```
