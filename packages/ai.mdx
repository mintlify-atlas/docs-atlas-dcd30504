# AI Package

The `@beztack/ai` package provides a pre-configured integration with the Vercel AI SDK and Amazon Bedrock, enabling you to build AI-powered features quickly.

## Installation

The AI package is optional. Add it during project creation or initialization:

```bash Terminal
pnpm dlx beztack init --modules ai
```

Or install it manually:

```bash Terminal
pnpm add @beztack/ai --filter @beztack/api
```

## Exports

The package re-exports everything from the Vercel AI SDK and provides a configured Bedrock provider:

### Bedrock Provider

```typescript
import { bedrock } from "@beztack/ai"
```

<ParamField path="bedrock" type="object">
  Pre-configured Amazon Bedrock provider instance
  
  **Configuration:**
  - **Region:** `us-east-1`
  - **Credentials:** Uses AWS credential provider chain (IAM roles, environment variables, profiles)
</ParamField>

### AI SDK Re-exports

All exports from `ai` package are available:

```typescript
import {
  generateText,
  streamText,
  generateObject,
  streamObject,
  embed,
  embedMany,
  tool,
  // ... and all other AI SDK exports
} from "@beztack/ai"
```

## Usage Examples

### Text Generation

```typescript Example
import { bedrock, generateText } from "@beztack/ai"

const { text } = await generateText({
  model: bedrock("anthropic.claude-3-5-sonnet-20241022-v2:0"),
  prompt: "Explain TypeScript generics in simple terms",
})

console.log(text)
```

### Streaming Responses

```typescript API Route
import { bedrock, streamText } from "@beztack/ai"
import { defineEventHandler } from "h3"

export default defineEventHandler(async (event) => {
  const result = streamText({
    model: bedrock("anthropic.claude-3-5-sonnet-20241022-v2:0"),
    prompt: "Write a haiku about coding",
  })

  return result.toDataStreamResponse()
})
```

### Object Generation

```typescript Structured Output
import { bedrock, generateObject } from "@beztack/ai"
import { z } from "zod"

const { object } = await generateObject({
  model: bedrock("anthropic.claude-3-5-sonnet-20241022-v2:0"),
  schema: z.object({
    recipe: z.object({
      name: z.string(),
      ingredients: z.array(z.string()),
      steps: z.array(z.string()),
    }),
  }),
  prompt: "Generate a cookie recipe",
})

console.log(object.recipe)
```

### Tool Calling

```typescript Agent with Tools
import { bedrock, generateText, tool } from "@beztack/ai"
import { z } from "zod"

const { text, toolCalls } = await generateText({
  model: bedrock("anthropic.claude-3-5-sonnet-20241022-v2:0"),
  tools: {
    weather: tool({
      description: "Get the weather in a location",
      parameters: z.object({
        location: z.string().describe("The location to get weather for"),
      }),
      execute: async ({ location }) => {
        // Fetch weather data
        return { temperature: 72, condition: "sunny" }
      },
    }),
  },
  prompt: "What's the weather in San Francisco?",
})
```

## Configuration

### AWS Credentials

The Bedrock provider uses the AWS SDK credential provider chain. Configure credentials using any of these methods:

<Note>
  The credential provider chain checks sources in this order:
  1. Environment variables (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`)
  2. IAM roles (for EC2, ECS, Lambda)
  3. AWS credential files (`~/.aws/credentials`)
  4. Container credentials
</Note>

**Environment Variables:**

```bash .env
AWS_ACCESS_KEY_ID=your_access_key
AWS_SECRET_ACCESS_KEY=your_secret_key
AWS_REGION=us-east-1
```

**IAM Role (Recommended for Production):**

No configuration needed - the SDK automatically uses attached IAM roles.

### Available Models

Amazon Bedrock supports multiple AI models:

- **Claude 3.5 Sonnet:** `anthropic.claude-3-5-sonnet-20241022-v2:0`
- **Claude 3 Opus:** `anthropic.claude-3-opus-20240229`
- **Claude 3 Haiku:** `anthropic.claude-3-haiku-20240307`
- **Titan Text:** `amazon.titan-text-express-v1`

<Tip>
  Check the [Amazon Bedrock documentation](https://docs.aws.amazon.com/bedrock/latest/userguide/models-supported.html) for the latest model IDs and regional availability.
</Tip>
