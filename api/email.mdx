The `@beztack/email` package provides type-safe email sending with React Email template support and Resend as the email provider.

## Installation

```bash
pnpm add @beztack/email
```

## Environment Variables

Configure Resend credentials in your environment:

```bash
RESEND_API_KEY=re_your_api_key
RESEND_FROM_EMAIL=noreply@yourdomain.com
RESEND_FROM_NAME=Your App Name
```

## Exports

## Advanced Usage

### Sending Custom HTML Emails

For custom emails not covered by templates, use the lower-level API:

```typescript
import type { SendEmailProps, EmailResult } from "@beztack/email"

// Not exported by default - import from lib/send
import { send } from "@beztack/email/lib/send"

const result: EmailResult = await send({
  to: "user@example.com",
  subject: "Custom Email",
  html: "<h1>Hello World</h1><p>Custom HTML content</p>",
})
```

### Sending with React Components

```typescript
import type { SendEmailPropsWithReact } from "@beztack/email"
import { sendWithReact } from "@beztack/email/lib/send"

const MyEmailTemplate = ({ name }: { name: string }) => (
  <div>
    <h1>Hello {name}</h1>
    <p>This is a custom React email</p>
  </div>
)

const result = await sendWithReact({
  to: "user@example.com",
  subject: "Custom React Email",
  react: <MyEmailTemplate name="John" />,
})
```

### Error Handling

```typescript
import { sendEmail } from "@beztack/email"

const result = await sendEmail({
  type: "welcome",
  to: "user@example.com",
  data: {
    username: "John Doe",
    loginUrl: "https://app.example.com/login",
  },
})

if (!result.success) {
  console.error("Failed to send email:", result.error)
  // Handle error (retry, log, alert admin, etc.)
} else {
  console.log("Email sent successfully:", result.data?.id)
}
```

### Using in API Routes

```typescript
import { defineEventHandler, createError } from "h3"
import { sendEmail } from "@beztack/email"

export default defineEventHandler(async (event) => {
  const { email, username } = await readBody(event)

  const result = await sendEmail({
    type: "welcome",
    to: email,
    data: {
      username,
      loginUrl: "https://app.example.com/login",
    },
  })

  if (!result.success) {
    throw createError({
      statusCode: 500,
      message: "Failed to send welcome email",
    })
  }

  return { success: true, emailId: result.data?.id }
})
```

## Type Definitions

### EmailType

Available email template types:

```typescript
type EmailType =
  | "welcome"
  | "password-reset"
  | "subscription-confirmation"
  | "organization-invitation"
```

### SendEmailUnifiedProps

Unified props for templated email sending:

```typescript
type SendEmailUnifiedProps = {
  type: EmailType
  to: string
  data: {
    username?: string
    loginUrl?: string
    resetUrl?: string
    planName?: string
    amount?: string
    billingPeriod?: string
    dashboardUrl?: string
    invitedByUsername?: string
    invitedByEmail?: string
    organizationName?: string
    invitationUrl?: string
  }
}
```

### EmailResult

Result object returned by all send functions:

```typescript
type EmailResult = {
  success: boolean
  data?: {
    id?: string
    message?: string
    [key: string]: unknown
  }
  error?: string
}
```

## Template Customization

Email templates are located in `packages/email/emails/` and use React Email components:

- `packages/email/emails/welcome.tsx` - Welcome email React component
- `packages/email/emails/password-reset.tsx` - Password reset React component
- `packages/email/lib/templates.ts` - HTML fallback templates

To customize templates, edit the React components or HTML templates directly.

## Testing Emails

During development, emails are sent through Resend. Use Resend's test mode or a service like [MailHog](https://github.com/mailhog/MailHog) for local testing.

```bash
# Set test API key in .env
RESEND_API_KEY=re_test_key
```
