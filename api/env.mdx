The `@beztack/env` package provides runtime environment variable validation with full TypeScript type inference for different application contexts (API, UI, Next.js).

## Installation

```bash
pnpm add @beztack/env
```

## Exports

The package exports environment configurations for different runtime environments:

- `@beztack/env/api` - API/Nitro server configuration
- `@beztack/env/ui` - Vite UI client configuration
- `@beztack/env/nextjs` - Next.js full-stack configuration

## API Environment (`@beztack/env/api`)

Server-side environment variables for Nitro API applications.

import { env } from "@beztack/env/api"

### Usage Example

```typescript
import { env } from "@beztack/env/api"
import { db } from "drizzle-orm/postgres-js"
import postgres from "postgres"

const client = postgres(env.DATABASE_URL)
export const database = db({ client })

console.log("Running in:", env.NODE_ENV)
console.log("Payment provider:", env.PAYMENT_PROVIDER)
```

### Runtime Validation

The API environment performs additional runtime validation:

```typescript
// Validates payment URLs are required and valid
if (!env.PAYMENTS_SUCCESS_URL && !env.POLAR_SUCCESS_URL) {
  throw new Error("Missing payment success URL")
}

// Validates Polar UUIDs when using Polar
if (env.PAYMENT_PROVIDER === "polar") {
  // All product IDs must be valid UUIDs
}

// Validates Mercado Pago token when using MP
if (env.PAYMENT_PROVIDER === "mercadopago") {
  // MERCADO_PAGO_ACCESS_TOKEN must be set
}
```

## UI Environment (`@beztack/env/ui`)

Client-side environment variables for Vite applications.

### Import

```typescript
import { env } from "@beztack/env/ui"
```

### Available Variables

<ParamField path="env.VITE_API_URL" type="string">
  API backend URL (defaults to "http://localhost:3000", must be valid URL)
</ParamField>

<ParamField path="env.VITE_BASE_PATH" type="string">
  Application base path (defaults to "/", optional)
</ParamField>

<ParamField path="env.VITE_PAYMENT_PROVIDER" type="'polar' | 'mercadopago'">
  Active payment provider (defaults to "polar")
</ParamField>

<ParamField path="env.VITE_SUBSCRIPTION_MODE" type="'user' | 'organization'">
  Subscription model (defaults to "organization")
</ParamField>

<ParamField path="env.VITE_MERCADO_PAGO_PUBLIC_KEY" type="string">
  Mercado Pago public key for client-side SDK
</ParamField>

### Usage Example

```typescript
import { env } from "@beztack/env/ui"

const API_BASE = env.VITE_API_URL

async function fetchUser() {
  const response = await fetch(`${API_BASE}/api/user`)
  return response.json()
}

console.log("Payment provider:", env.VITE_PAYMENT_PROVIDER)
```

### Client Prefix Requirement

All UI environment variables **must** be prefixed with `VITE_` to be exposed to the client:

```bash
# .env
VITE_API_URL=https://api.example.com  # ✅ Accessible in browser
API_SECRET=secret123                  # ❌ NOT accessible in browser
```

## Next.js Environment (`@beztack/env/nextjs`)

Full-stack environment configuration for Next.js applications.

### Import

```typescript
import { env } from "@beztack/env/nextjs"
```

### Available Variables

<Accordion title="Server-side">
<ParamField path="env.NODE_ENV" type="'development' | 'production' | 'test'">
  Node.js environment (defaults to "development")
</ParamField>
</Accordion>

<Accordion title="Client-side">
Client variables must be prefixed with `NEXT_PUBLIC_`:

```typescript
// Add your client variables to the schema:
client: {
  NEXT_PUBLIC_API_URL: z.string().url(),
}
```
</Accordion>

### Usage Example

```typescript
import { env } from "@beztack/env/nextjs"

// Server-side only
export async function getServerSideProps() {
  console.log("Running in:", env.NODE_ENV)
  return { props: {} }
}

// Client-side
export default function Page() {
  // Access NEXT_PUBLIC_ prefixed variables
  return <div>Environment: {env.NODE_ENV}</div>
}
```

## Type Safety

All environment configurations provide full TypeScript type inference:

```typescript
import { env } from "@beztack/env/api"

// ✅ TypeScript knows this is a string
const dbUrl: string = env.DATABASE_URL

// ✅ TypeScript knows this is "polar" | "mercadopago"
const provider: "polar" | "mercadopago" = env.PAYMENT_PROVIDER

// ❌ TypeScript error - property doesn't exist
const invalid = env.NONEXISTENT_VAR
```

## Error Handling

Missing required variables throw errors at startup:

```typescript
// If DATABASE_URL is not set:
// Error: Missing environment variable: DATABASE_URL

// If VITE_API_URL is not a valid URL:
// Error: Invalid environment variable VITE_API_URL: must be a valid URL
```

## Best Practices

### Use Environment-Specific Imports

```typescript
// ✅ In API code
import { env } from "@beztack/env/api"

// ✅ In UI code
import { env } from "@beztack/env/ui"

// ❌ Don't mix environments
import { env } from "@beztack/env/api" // in UI code
```

### Never Expose Secrets to Client

```typescript
// ✅ Server-only secrets (no VITE_ or NEXT_PUBLIC_ prefix)
BETTER_AUTH_SECRET=secret123
RESEND_API_KEY=re_secret

// ✅ Client-safe values (with prefix)
VITE_API_URL=https://api.example.com
NEXT_PUBLIC_APP_NAME=My App
```

### Validate URLs and UUIDs

The package automatically validates:
- URLs using Zod's `.url()` validator
- UUIDs using Zod's `.uuid()` validator
- Email addresses using Zod's `.email()` validator
