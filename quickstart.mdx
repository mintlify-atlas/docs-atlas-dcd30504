Get your Beztack project up and running in minutes. This guide will walk you through creating a new project, setting up the environment, and starting the development servers.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js 20+** - [Download here](https://nodejs.org/)
- **pnpm 9+** - Install with `npm install -g pnpm`
- **PostgreSQL** - [Download here](https://www.postgresql.org/download/)

<Note>
  You can verify your installations by running `node --version`, `pnpm --version`, and `psql --version`.
</Note>

## Create Your Project

<Steps>
  <Step title="Run the create command">
    Use pnpm to create a new Beztack project:

    <CodeGroup>
    ```bash pnpm
    pnpm create beztack my-app
    ```

    ```bash npm
    npm create beztack my-app
    ```

    ```bash yarn
    yarn create beztack my-app
    ```
    </CodeGroup>

    The CLI will prompt you to:
    - Enter a project name
    - Enter a project description
    - Select optional modules (payments, email, AI, OCR)
    - Choose a payment provider (Polar or Mercado Pago) if payments is enabled
    - Decide whether to initialize git
    - Decide whether to install dependencies
  </Step>

  <Step title="Navigate to your project">
    ```bash
    cd my-app
    ```
  </Step>

  <Step title="Set up environment variables">
    Copy the example environment file:

    ```bash
    cp .env.example .env
    ```

    Edit `.env` with your configuration:

    ```bash .env
    # Database
    DATABASE_URL="postgresql://postgres:postgres@localhost:5432/beztack"

    # Auth
    BETTER_AUTH_SECRET="your-secret-here"  # Generate with: openssl rand -base64 32
    BETTER_AUTH_URL="http://localhost:3000"

    # Email (optional)
    RESEND_API_KEY="your-resend-api-key"

    # Payments (optional - based on your selection)
    POLAR_ACCESS_TOKEN="your-polar-token"
    ```

    <Warning>
      Always generate a strong `BETTER_AUTH_SECRET` using `openssl rand -base64 32`. Never commit real secrets to version control.
    </Warning>
  </Step>

  <Step title="Set up the database">
    Create your PostgreSQL database and run migrations:

    ```bash
    # Push schema to database (development)
    pnpm push

    # OR generate and run migrations (production)
    pnpm migrate
    ```

    <Tip>
      Use `pnpm push` for development to quickly sync schema changes. Use `pnpm migrate` for production to generate and apply migrations.
    </Tip>
  </Step>

  <Step title="Start the development servers">
    Start all apps in development mode:

    ```bash
    pnpm dev
    ```

    This will start:
    - **Frontend (UI)**: http://localhost:5173
    - **Backend (API)**: http://localhost:3000
    - **Docs**: http://localhost:3002
  </Step>

  <Step title="Verify installation">
    Open your browser and navigate to:

    - http://localhost:5173 - Main application
    - http://localhost:3000/api/health - API health check
    - http://localhost:3002 - Documentation site

    You should see the Beztack welcome screen!
  </Step>
</Steps>

## CLI Options Reference

The `beztack create` command supports the following flags for non-interactive mode:

| Flag | Description |
|------|-------------|
| `--yes`, `--non-interactive` | Run without prompts |
| `--name <project-name>` | Set project name |
| `--description <text>` | Set project description |
| `--git` / `--no-git` | Enable/disable git initialization |
| `--install` / `--no-install` | Enable/disable dependency installation |
| `--init` / `--no-init` | Enable/disable module configuration |
| `--modules <list>` | Comma-separated optional modules |
| `--payment-provider <provider>` | Payment provider: `polar` or `mercadopago` |
| `--template-source <path>` | Custom template source (for local development) |

Example:

```bash
pnpm create beztack my-app --yes --modules payments,email --payment-provider polar
```

## Common Commands

```bash
# Start all apps
pnpm dev

# Build all apps
pnpm build

# Run linting
pnpm lint

# Run database migrations
pnpm migrate

# Push schema changes (dev only)
pnpm push

# Start a specific app
npx nx dev ui
npx nx dev api
npx nx dev docs
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port already in use">
    If you see errors about ports being in use:

    ```bash
    # Find and kill process on port 5173 (UI)
    lsof -ti:5173 | xargs kill -9

    # Find and kill process on port 3000 (API)
    lsof -ti:3000 | xargs kill -9
    ```
  </Accordion>

  <Accordion title="Database connection errors">
    Ensure PostgreSQL is running and your `DATABASE_URL` is correct:

    ```bash
    # Check PostgreSQL status
    pg_isready

    # Test connection
    psql $DATABASE_URL -c "SELECT version();"
    ```
  </Accordion>

  <Accordion title="Module not found errors">
    If you see module resolution errors:

    ```bash
    # Clear node_modules and reinstall
    rm -rf node_modules
    pnpm install

    # Clear NX cache
    npx nx reset
    ```
  </Accordion>
</AccordionGroup>
